连接查询
基本概念
	就是将两个或两个以上的表,"连接起来",当做一个数据源，并从中去取得所需要的数据
到底怎么链接？
	将每一个表的每一行数据两两之间相互对接起来,每次对接的结果都是连接结果的"一行"数据，
另一种导入 登陆后 source "文件路径"
select * from join1,join2;#这种没有条件的链接

select * from 表1,表2
select *　from　表1　join　表２；
select * from 表1 cross join 表2;
select * from 表1 inner join 表2;
他们都叫交叉连接,其实就是没有条件的链接

连接的基本形式
在代码级别，连接的基本形式：
表1 [连接形式] join 表2 [on 连接条件]
表1 [连接形式] join 表2 [on 连接条件] join 表3 [on 连接条件]....
更多表，依次类推

连接分类
交叉连接
	刚才连接的基本概念的时候的连接形式(结果)--它没有条件 "笛卡尔积"
		有n1+n2个列
		有m1*m2个行
内连接
	形式：select * from 表1 [inner] join 表2 on 连接条件
	表1.ID=表2.ID被称为"连接条件",它基本上就是我们之前所学的"外键"关系的描述
注意:这种的表跟表之间的内链接查询,虽然可以提现为表跟表之间的关系--通常就是外键关系--但并不是有外键关系才能使用。

左（外）连接 left(outer)join
	形式：表1(左表) left(outer)join 表2(右表) on 连接条件
	含义:其实就是将两个表的内链接的结果,再加上左边表的不符合内链接所设定的条件的那些数据的结果
	可见左边表的内容会全部取出
右（外）连接 right(outer)join
	形式：表1(左表) right(outer)join 表2(右表) on 连接条件
	含义:其实就是将两个表的内链接的结果,再加上右边表的不符合内链接所设定的条件的那些数据的结果
	可见右边表的内容会全部取出
全（外）连接 full(outer)join
	形式：没有形式,因为mysql不支持全链接的语法
	含义:其实就是将两个表的内链接的结果,再加上左边表的不符合内链接所设定的条件的那些数据的结果
	再加上右边表的不符合内链接所设定的条件的那些数据的结果
	
子查询
	select里面还有select
	标量子查询  一个子查询的结果是一行一列只有一个值
	行子查询    一个子查询的结果是一行多列此时可以当做一行来使用通常放在"行比较语句"中
			类似 where row(字段1，字段2)=set(select 行子查询) 一一对应的比较
	列子插叙	一个子查询的结果是多行一列的时候此时可以当作多个值(5,17,8,22)
	表子查询	一个子查询的结果是多行多列当做一个表放在from后面
	
	作为主查询的结果数据:
	select c1,(select f1 from tab2) as f11 from tab1;#这里子查询应该只有一个数据(一行一列 标量子查询)
	作为主查询的条件数据:
	select c1 from tab1 where c1 in (select f1 from tab2) ;#这里子查询可以是多个数据(多行一列 列子查询)
	作为主查询的来源数据:
	select c1 from (select f1 as c1,f2 from tab2) as t2 ;#这里子查询可以是任意查询结果(表查询)
	
	总上：select 字段/表达式/子查询 [as 别名] from 表名/链接结果/子查询结果 where 字段/表达式/子查询的条件判断
	
常用的子查询
	比较运算符中的子查询
	形式：操作数 比较运算符 (标量子查询)
	说明：操作数,其实就是比较运算符的2个数据之一而已,通常就是一个字段；
	select ... from XXX where id>5
	举例:
		select * from product  where price =(select max(price)) from product);
	
	使用in的子查询
	以前用的in的用法：
		XX in (值1,值2,值3,...);
	则in子查询为:
		操作数 in(列子查询)
		
	使用any的子查询
	形式:操作数 比较运算符 any(列子查询)
	含义:
		当某个操作符(字段)对于该子查询的其中任意一个值,都满足该比较运算符,则就满足了条件;
		即有一个满足就算
		selec * from tab1 where id > any(select f1 from tab2)
		
	使用all的子查询
	形式:操作数 比较运算符 all(列子查询)
	含义:
		当某个操作符(字段)对于该列子查询的所有数据值,都满足该比较运算符,才算满足了条件;
		即要求全部满足，才算满足
		selec * from tab2 where f1 > all(select f1 from tab1)
	
	使用some的子查询
	一句话:some是any同义词
	
	使用exists的查询
	形式:
		where exists(子查询)
	含义:
		该子查询如果有数据,则exists的结果是true,否则就是false
		
union联合查询
	数量一样 select 语句1 ...union select 语句2
	两个select语句的查询结果的字段是必须一致
	最好两个查询语句的字段类型也一致性；
	联合查询的语句,默认会"自动消除重复行"即默认是distinct
	如果想要将所有数据的显示(允许重复行),就使用all
